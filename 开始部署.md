### 方式二：手动部署

#### 步骤 1：推送代码到 GitHub

```bash
cd c:\Users\Administrator\新建文件夹\888

# 添加远程仓库（替换你的用户名）
git remote add origin https://github.com/你的用户名/ethusdt-quantitative-system.git

# 推送代码
git push -u origin master
```

#### 步骤 2：部署到 Streamlit Cloud

1. 访问：https://share.streamlit.io/
2. 点击 "Sign in with GitHub"
3. 授权登录
4. 点击 "New app"
5. 填写信息：
   - Repository: ethusdt-quantitative-system
   - Branch: master
   - Main file path: app_coinex.py
6. 点击 "Deploy!"

#### 步骤 3：配置 API Key

在 Streamlit Cloud 应用中：
1. Settings → Secrets
2. 添加：

```toml
COINEX_ACCESS_ID = "ck_ffiiw9tghpmo"
COINEX_SECRET_KEY = "your_secret_key_here"
```

3. Save

## ⚙️ 高级配置（可选）

### 1. 使用在线 AI API

修改 `app_coinex.py`，添加：

```python
import openai

def get_ai_analysis_online(prompt):
    client = openai.OpenAI(
        api_key=st.secrets.get("DEEPSEEK_API_KEY"),
        base_url="https://api.deepseek.com"
    )
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3
    )
    return response.choices[0].message.content
```

## 🐛 故障排除

### 问题 1：推送失败

**解决方案**：
```bash
# 配置 Git 用户信息
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

# 使用 Personal Access Token
# GitHub → Settings → Developer settings → Personal access tokens
```

### 问题 2：部署失败

**检查清单**：
- [ ] GitHub 仓库是否为 Public
- [ ] requirements.txt 是否正确
- [ ] Python 文件路径是否正确

### 问题 3：API Key 无效

**解决方案**：
```bash
# 检查 Secrets 格式
COINEX_ACCESS_ID = "ck_ffiiw9tghpmo"  # 注意引号
COINEX_SECRET_KEY = "your_key"        # 不要有多余空格
```

### 问题 4：AI 功能不工作

**原因**：Streamlit Cloud 不支持 Ollama

**解决方案**：
- 使用在线 API（DeepSeek/OpenAI）
- 或在应用中禁用 AI
